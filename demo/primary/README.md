# Muchlinski Civil War Prediction — broken reproduction

A small synthetic reproduction of:

> Muchlinski, D., Siroky, D., He, J., & Kocher, M. (2016).
> *Comparing Random Forest with Logistic Regression for Predicting
> Class-Imbalanced Civil War Onset Data.* Political Analysis.

This fixture exists to demonstrate **Paper Trail** finding and fixing a documented data-leakage bug.

## The claim

A Random Forest classifier substantially outperforms logistic regression at predicting civil war onset from country-year panel data.

## The bug (intentional)

In [`src/prepare_data.py`](src/prepare_data.py), a `KNNImputer` is fit on the **entire dataframe before the train/test split**, and — critically — the target column (`civil_war_onset`) is still in the dataframe when the imputer runs. KNN nearest-neighbor imputation therefore uses the target as one of the features when filling in missing values, then the target is re-extracted afterward. This is a compound leakage bug:

1. **Imputation-before-split:** imputer sees test-set feature values.
2. **Target leakage via imputation:** target column is used as a feature during imputation.

## Expected metric delta

Using a deterministic 75/25 stratified split (`random_state=42`), ROC-AUC on the held-out test set:

|                      | Random Forest | Logistic Regression |
|----------------------|:-------------:|:-------------------:|
| Broken (ship default) | **0.9562**    | **0.8091**          |
| Fixed (correct prep)  | 0.9070        | 0.6962              |
| Delta                | −0.049        | −0.113              |

The LR baseline was **inflated by 11 percentage points** by the leakage. After the fix, it's clear that LR is a substantially weaker baseline than the repo's headline numbers suggested.

## The fix

Drop the target column *and* the ID columns before imputation, split first, then fit `KNNImputer` on training features only:

```python
y = df[TARGET]
X = df.drop(columns=[TARGET, *ID_COLS])
X_train_raw, X_test_raw, y_train, y_test = train_test_split(
    X, y, test_size=test_size, random_state=RANDOM_STATE, stratify=y,
)
imputer = KNNImputer(n_neighbors=5)
X_train = pd.DataFrame(imputer.fit_transform(X_train_raw), columns=X.columns, index=X_train_raw.index)
X_test  = pd.DataFrame(imputer.transform(X_test_raw),     columns=X.columns, index=X_test_raw.index)
```

## Dataset

`data/civilwar.csv` is synthetic, deterministic, 3,000 country-year observations with ~9% civil-war-onset prevalence. Generated by [`data/generate.py`](data/generate.py) with `RANDOM_STATE=42`.

Key structural property: **MNAR missingness** — rows where `civil_war_onset == 1` have ~3× higher missingness across numeric features, simulating disrupted data collection in conflict zones. This is what makes the leakage quantitatively meaningful.

Features:

| Column | Type | Notes |
|---|---|---|
| `country_id` | int | identifier |
| `year` | int | identifier |
| `gdp_per_capita` | float | has missing values |
| `gdp_growth_rate` | float | has missing values |
| `population_log` | float | has missing values |
| `prior_war` | int | no missing |
| `ethnic_frac` | float | has missing values |
| `mountain_terrain` | float | no missing |
| `oil_exports_pct` | float | has missing values |
| `polity_score` | float | has missing values |
| `neighbor_war` | int | no missing |
| `years_since_war` | int | has missing values |
| `arms_imports` | float | has missing values |
| `infant_mortality` | float | has missing values |
| `primary_education_ratio` | float | has missing values |
| `civil_war_onset` | int | **target** (0/1) |

## Notes

- Data is synthetic. No real civil war data is used here.
- The coefficients in `data/generate.py` were tuned so that (a) RF has something real to learn (AUC > 0.70 even on clean data), and (b) the leakage bug produces a visible, dramatic inflation that the agent can demonstrate in a before/after metric comparison.
- Seed `42` is intentional — change it only if the agent's first-pass hypothesis ranking becomes seed-dependent.
