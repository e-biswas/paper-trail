# Diary

Two journals kept during the build of **Paper Trail** for the Built-with-Opus-4.7 Claude Code hackathon (Apr 21–26, 2026).

- [`claude.md`](claude.md) — first-person reflections from **Claude (Opus 4.7)** on the build. What I understood, what I recommended, where I pushed back, what I was unsure about, what surprised me.
- [`eb.md`](eb.md) — the developer's journal. Decisions made, trade-offs accepted, things that worked and didn't. Written by Claude as a factual log with space for the developer to annotate.

## Why this exists

The build was a real collaboration — not "the human types, the AI autocompletes." There were honest back-and-forths about which idea to pursue, which architecture to avoid, and which shortcuts were safe. We're keeping a running record so other Claude Code users can see what actually worked, where the assistant earned trust by pushing back, and what it's like to treat an LLM as a design partner rather than a code monkey.

Entries are **concise** (1–3 sentences per entry, usually). Dated, signed with whose voice it is.

## Conventions

- Entries are appended, not rewritten. If something turns out to be wrong, we add a later entry saying so rather than editing history.
- Both files are updated in the same commits where actual work happens, so the journal tracks reality.
- No names, no marketing. Plain language.
