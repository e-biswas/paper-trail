This fixture exists to demonstrate **Paper Trail** finding and fixing a well-documented duplicate-image leakage bug.

## The claim

A Random Forest classifier trained on image-level handcrafted features + clinical metadata can identify melanoma with ROC-AUC > 0.70.

## The bug (intentional)

The dataset contains ~525 duplicate images (same `image_hash`, same features, same target, different `image_name`). In [`src/prepare_data.py`](src/prepare_data.py), `train_test_split` is applied directly to the dataframe **without first deduplicating by `image_hash`**. Duplicates leak between the train and test folds; Random Forest memorizes images it sees during training and recognizes the same hashes as test examples.

This mirrors the 400+ exact duplicates the ISIC 2020 Challenge organizers documented in the public training set â€” see community GitHub issues on the challenge repo.

## Expected metric delta

With `random_state=42`, 80/20 stratified split, 300-tree Random Forest:

|                      | ROC-AUC |
|----------------------|:-------:|
| Broken (ship default) | **0.7153** |
| Fixed (dedup first)   | 0.6522     |
| Delta                 | **âˆ’0.063** |

The paper's headline metric (AUC â‰ˆ 0.72) **vanishes** once duplicates are removed: honest performance is closer to 0.65.

## The fix

Before calling `train_test_split`, drop duplicate rows by `image_hash`:

```python
df = df.drop_duplicates(subset=["image_hash"], keep="first").reset_index(drop=True)
```

(A stricter fix deduplicates by perceptual hash â€” i.e., near-duplicates, not just exact. MVP uses exact-hash dedup because the synthetic dataset ships with exact duplicates only.)

## Dataset

`data/isic_metadata.csv` is synthetic, deterministic, 4,025 rows (3,500 unique hashes + 525 duplicates) with 12.0% melanoma prevalence. Generated by [`data/generate.py`](data/generate.py) with `RANDOM_STATE=42`.

Schema (matches real ISIC 2020 metadata):

| Column | Type | Notes |
|---|---|---|
| `image_name` | str | unique per row (`ISIC_NNNNNNN` or `ISIC_DUP_NNNNNNN`) |
| `image_hash` | str | **NOT unique** â€” duplicates share a hash |
| `patient_id` | str | `IP_NNNNN` |
| `age_approx` | int | |
| `sex` | int | 0/1 |
| `anatom_site_general_challenge` | str | torso / head_neck / â€¦ |
| `mean_intensity`, `std_intensity`, `skewness`, `kurtosis` | float | image statistics (synthetic) |
| `red_mean`, `green_mean`, `blue_mean` | float | color channels |
| `edge_density`, `symmetry_score`, `size_pixels` | float/int | shape / size |
| `target` | int | **target** â€” 0 benign, 1 melanoma |

Class separation is deliberately weak (features overlap substantially across classes). Clean-data AUC sits around 0.65; the duplicate leakage drives it up to 0.72 because RF memorizes training hashes.
