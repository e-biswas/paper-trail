## claim_tested

The paper claims that, for discrete diffusion language models (DLMs) trained on Nemotron-CC with CompleteP-style width/depth scaling, per-token compute-optimal behavior differs sharply across noise types: uniform diffusion prefers more parameters and fewer tokens than masked diffusion, and the fitted scaling laws extrapolate to held-out 3B (masked + uniform) and 10B (uniform) models that report train-PPL values of 11.3, 11.7, and 9.15 respectively. These numbers are the headline reproducibility targets in Table / Fig. 1 of the paper.

## evidence_gathered

- Entry points: `main.py`, `main_gpu.py`, `main_ray.py` all import `easydel`/`jax`; evaluation is `eval_ray.py` (`ray.init()` at module load, TPU-only) wrapping `lm_eval.__main__.cli_evaluate`. No lightweight local entrypoint.
- Core model/loss/schedule implementations reviewed: `gidd_easydel/diffusion_trainer/loss.py` (GiddLoss), `gidd_easydel/diffusion_trainer/schedule.py` (MixingSchedule, HybridMixingDistribution, LinearMixingRate, `get_loss_weights`, `get_elbo_weights`), `gidd_easydel/diffusion_trainer/_fn.py` (`prepare_batch`, `compute_loss`, `training_step`), `gidd_easydel/likelihood.py` (ELBO integrator), `gidd_easydel/diffusion_trainer/diffusion_trainer.py` (wires tokenizer.mask_token_id into GiddLoss).
- Latent-default observation: `loss.py:27` declares `mask_token_id: int = -1`; `loss.py:85` uses it as a positive-axis index to zero the corresponding logit. Callable only via `DiffusionTrainer.__init__` (`diffusion_trainer.py:76`) which overrides to `tokenizer.mask_token_id`. Latent but unused in the reference runs.
- Likelihood path observation: `likelihood.py:155–163` drops `hybrid_scale=`. Paper fixes `a = 1` so default (`hybrid_scale=1.0`) matches; not a defect for the reference runs.
- Data pipeline: `shuffle_dataset.py` and `gidd_easydel/data.py` shuffle at partition + reservoir-buffer granularity with no explicit duplicate-detection step. The pre-tokenized dataset (`dvruette/gidd-nemotron-cc-pretok`) is externally hosted; dedup handling was upstream and cannot be audited from this repo.
- Local environment: `/Users/eb/Downloads/experiments/paper-trail/.venv/bin/python` lacks `jax` and `flax`; `pytest` is also unavailable. No subset of the training or inference code is runnable in this sandbox.

## root_cause

Not determined. The investigation was bounded by environment: none of the five candidate hypotheses could be discriminated because (1) no fixture eval script exists in the repo, and (2) the JAX/flax runtime required by every model-touching file is absent from the designated interpreter. All five hypotheses remain at or below 0.35 confidence after static inspection; none reach the 0.85 verdict threshold.

## fix_applied

None. Per the operating contract, a fix must be paired with an executable `## Metric delta:` that shows `after != before`. This sandbox can produce neither the "before" nor the "after" metric for any non-trivial reproducibility claim in this repo, so no edit was attempted. Running an unverified patch would risk silently regressing the paper's numbers without any safety net.

- Whether the uniform-in-α integration grid at `likelihood.py:167–175` produces a biased ELBO when combined with `elbo_weights / p_log_snr.clip(1e-12)`; I was unable to run the likelihood function on any toy tensor to check the empirical bias against an analytic reference.
